# Skill: New project — `set up a project`

Turn the pre-built **`(PROJECT TEMPLATE)`** into a real, live project — no folder-making or copy-paste by the user.

> **Hard rule:** NEVER edit `03 Projects/(PROJECT TEMPLATE)/` directly. Always **copy** it.

## Steps
1. **Ask three things:**
   - What's the project called?
   - What does "done" look like?
   - Roughly how does work flow from idea → done? *(A simple default is fine: `00 Ideas → 01 In Progress → 02 Done`.)*
2. **Copy** `03 Projects/(PROJECT TEMPLATE)/` to `03 Projects/<Project Name>/` — including its `00 Ideas/`, `01 In Progress/`, and `02 Done/` subfolders.
3. **Fill in** the new project's `CLAUDE.md` from the answers: name + one-line purpose, Claude's role + prime directive, the process, and a "just created" status. **Remove the "this is a template" warning line at the top.**
4. **Register it:** add a bullet for the project under `## My Current Projects` in the root `CLAUDE.md`.
5. Tell the user it's ready and where to drop their first idea (`00 Ideas/`).
