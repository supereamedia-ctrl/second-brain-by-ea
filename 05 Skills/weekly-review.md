# Skill: Weekly review — `run my weekly review`

Keep the whole system current. This is where the compounding happens — a second brain you never update goes stale; one you review weekly gets smarter over time.

## Steps
1. **Read** the root `CLAUDE.md`, each project's `CLAUDE.md`, and `GOALS.md`.
2. **Ask** the user:
   - What's working this week?
   - What's not working?
   - What are you sitting on / need to decide?
   - Any deadlines or time-sensitive things coming up?
3. **Update:**
   - The `## Weekly Update` block in the root `CLAUDE.md` (refresh **Last updated** to today's date).
   - Progress and any changed numbers in `GOALS.md`.
   - *(Ask before editing anything the user wrote themselves.)*
4. **Write a dated review note** in `04 Reviews/Weekly Reviews/`, named with the `(C)` prefix — e.g. `(C) 2026-07-01 Weekly Review.md` — summarizing wins, misses, and next week's focus.
5. Close with the single most important thing to focus on next week.
